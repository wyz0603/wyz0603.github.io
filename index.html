关于我

我是吴杨智，电子信息学院电子信息工程专业的学生。大学期间，我专注于机器人操作系统（ROS / ROS2）的研究与应用，积累了丰富的实践经验。

我擅长 SLAM 建图、路径规划与多传感器融合，能够独立完成从底盘搭建、传感器集成到导航算法实现的全流程开发。曾在乐聚机器人软件部实习，参与人形机器人「夸父 KUAVO」上位机 Linux 应用的开发与维护，热衷于将算法落地到真实的机器人平台。

在多项国家级机器人赛事中获奖，享受用代码和硬件解决实际问题的过程。

人形机器人 MID360 激光雷达仿真与自主导航系统集成

基于 Leju Kuavo 人形机器人，在 Gazebo 仿真环境中打通「激光雷达仿真 → SLAM 建图 → 自主导航」全链路，并将该能力从开发分支迁移、集成到目标工程的 SLAM / 导航工作空间（ROS Noetic / catkin）。

🔗
仿真–真机数据桥接节点

编写 C++ 节点 livox_sim_bridge，将 Gazebo Livox 插件输出的点云实时转换为 FAST-LIO 所需的 CustomMsg 与导航用 PointCloud2，并对 IMU / 点云施加安装姿态补偿，打通「仿真传感器 → LIO 里程计 → octomap → move_base」数据流。

🧭
TF 树关键缺陷定位与修复

排查出 Gazebo Livox 插件重复发布 TF 导致 livox_link 出现双父节点、变换查询返回错误旋转，改由 robot_state_publisher 经 URDF 链路统一提供变换，解决下游建图 / 导航坐标错乱。

⚙️
导航参数系统性调优

修复 costmap_2d 障碍层命名空间配置缺陷（此前障碍层始终为空）；调整局部代价地图尺寸、膨胀半径与代价缩放因子，按前瞻公式重算 MPC 速度与时域参数，显著改善卡死与绕障。

🎯
运动控制与跨工程迁移

开发 Pure Pursuit 纯追踪控制 + 反应式重规划节点（路径被占据超阈值触发 clear_costmaps + 重规划）；在不同 git 分支工程间无损迁移，并通过 catkin build 编译验证全部产物。
